# Smart City Complaint Prioritization and Performance Analytics System

## Problem Statement
Traditional civic grievance systems process complaints in a first-come-first-served manner, without considering severity, urgency, or the number of citizens affected. This often delays the resolution of critical public issues and limits transparency in evaluating governance performance.

## Proposed Solution
This project aims to build an intelligent web-based system that allows citizens to submit mandatory geo-tagged complaints with image evidence. Using Machine Learning and Natural Language Processing, complaints are automatically classified and prioritized based on severity, repetition, and time pending. The system also provides analytics dashboards to evaluate administrative efficiency.

## Key Features
- Citizen complaint registration with mandatory geo-location and image upload
- NLP-based complaint classification
- ML-based priority scoring of complaints
- Admin dashboard for priority-based complaint management
- Governance performance analytics and visualization
- Role-based authentication and access control

## Tech Stack
**Frontend:** React.js, HTML, CSS, JavaScript  
**Backend:** Node.js, Express.js (REST APIs)  
**Database:** MongoDB  
**Machine Learning & NLP:** Python, Scikit-learn, TF-IDF  
**Analytics & Visualization:** Chart.js / Recharts  
**Authentication:** JWT  
**Version Control:** Git & GitHub
